zTree v2.6 Files VS v3.0 Files

  • 1、JS Files

    • zTree v2.6 have only one js file, but with the additional features, the file is also growing.
      zTree v3.0 was split by function, there are three js files. One core pack ( jquery.ztree.core-3.0.js ), two expansion packs -- checkbox / radio pack( jquery.ztree.excheck-3.0.js ) & edit pack ( jquery.ztree.exedit-3.0.js )
    • To use zTree v3.0, you must load core pack, and loaded the expansion pack as needed. This structure is conducive to develop zTree features or tools, without affecting the basic function.
    • In use, recommend to merger 'expansion pack' and 'core package' into a js file, save network resources. When merge js, please ensure to the core package of code is on the front.

  • 2、Css Files

    • zTree v2.6 have 2 css files. ( zTreeStyle.css & zTreeIcons.css )
      zTree v3.0 have only one file. ( zTreeStyle.css ), in order to avoid conflict of css style, the className of zTree's container have changed into 'ztree'.
    • In zTree v3.0, the main DOM (e.g. li, ul, + / - icon, a)'s className attribute increase the text about level, so you can set independent style for different levels.
    • In zTree v3.0, modify some rules about className, can better control the css style, especially the custom icon 'iconSkin' can be the perfect support IE6.

  • 3、Image Files

    • zTree v2.6: Icon file too, need loaded separately, is not conducive to optimizing
      zTree v3.0: The main icons are merged into one file. Use the 'CSS Sprites' technology to show icon
    • In order to avoid the 'connect line' spacing 1 pixel problem, in zTree v3.0, '+/-' icon retained 18 * 18 px, all the other icons change to 16 * 16 px. Please note: To upgrade zTree v3.0, you need to modify the icons size
