zTree v3.x Note


  •     FAQ is adapted based on the FAQ of v2.6. It doesn't cover everything about v3.x yet.I will gradually improve it in the future.

  • 1. JS Files

    • zTree v3.x: There are three js files, One core pack ( jquery.ztree.core-3.x.js ), two expansion packs -- checkbox / radio pack( jquery.ztree.excheck-3.x.js ) & edit pack ( jquery.ztree.exedit-3.x.js )
    • To use zTree v3.x, you must load core pack, and loaded the expansion pack as needed. This structure is conducive to develop zTree features or tools, without affecting the basic function.
    • In use, recommend to merger 'expansion pack' and 'core package' into a js file, save network resources. When merge js, please ensure to the core package of code is on the front.

  • 2. Css Files

    • zTree v3.x have only one file. ( zTreeStyle.css ), in order to avoid conflict of css style, the className of zTree's container have changed into 'ztree'.
    • In zTree v3.x, the main DOM (e.g. li, ul, + / - icon, a)'s className attribute increase the text about level, so you can set independent style for different levels.

  • 3. Image Files

    • zTree v3.x: The main icons are merged into one file. Use the 'CSS Sprites' technology to show icon
    • In order to avoid the 'connect line' spacing 1 pixel problem, in zTree v3.x, '+/-' icon retained 18 * 18 px, all the other icons change to 16 * 16 px. Please note: To upgrade zTree v3.x, you need to modify the icons size
    • In v3.x demo, custom icon is a separate image file, the actual development proposal also use 'CSS Sprites'.

  • 4. zTree Node Data

    • zTree use JSON data, please carefully view the API documentation "treeNode data details", and the demo's source code.

  • 5. Debugging

    • Recommend the use of chrome or firefox to debug. Personal recommendation: chrome developer tools or firefox plug-ins 'firebug'.
